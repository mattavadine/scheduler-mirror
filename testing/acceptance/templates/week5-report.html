  • Fill in the department name where "X" appears in the title above.

  • Under the heading "Open an Existing Schedule, Create a New Schedule", describe any problems that arise when opening a new schedule or creating a new schedule. Use the newly created empty schedule for the rest of the testing.

  • Under the heading "Enter Resource Data", describe any problems that arise when entering database records for instructors, courses, or locations. If there were no problems, say "All data successfully entered." If there were one or more problems, include links to the Jira issues that were posted for the problems. Before posting a Jira issue, search the existing issue list and do not re-post issues for duplicate problems.

  • Under the heading "Generate a Schedule", describe the quality of the schedule that was automatically generated, in particular:
    1. Did it contain all classes?
    2. Were they at reasonable times?
    3. Were there any scheduling conflicts?


    If the exact Winter 2012 was generated directly, say "Winter 2012 schedule successfully generated, with no edits." If there were one or more problems, include links to the Jira issues that were posted for the problems.

    NOTE: If you want to force the generation of a correct schedule, you can try to enter specific instructor preferences to ensure that one instructor gets exactly the classes that are scheduled for that instructor in the Winter 2012. It would be better to enter preferences more broadly, where the actual Winter 2012 times as courses are within the preference range of the instructor who got them.
  • Under the heading "Edit the Schedule", describe how close you were able to get to the actual Winter 2012 schedule for the department, using the Scheduler program editing functionality. If there were no problems, say "Winter 2012 schedule successfully produced." If there were one or more problems, include links to the Jira issues that were posted for the problems.

Include links to any Jira issues that were posted for the problems.

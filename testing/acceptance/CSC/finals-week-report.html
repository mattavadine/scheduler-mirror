Finals Week Acceptance Test Report for the CSC Department



1. Create, Save, and Re-Open a New Schedule

    On the Login Page:

    1. With the top-most banner gone, which I like, we need a message identifying the program, above or in place of the 'Login' label . I suggest changing the label 'Logon' to 'Login to the Scheduling Program'

    2. Need 10 to 20 pixels of padding between the text edit box and 'Login' button.

    3. Need some roll-over help on all of the widgets saying "If you need a login ID, please contact your department scheduler." We need to think about how this relates to CAS authentication. Specifically, once someone authenticates through CAS, we then need to further filter to check that they are allowed access to the scheduler program. Again, this needs further thought.

    4. Need to have the 'Return' (aka, 'Enter') key be a keyboard shortcut for pressing the 'Login' button.

    On the 'My Scheduling Documents' page:

    1. At present, there are two check box lines with empty document names. This is may be an artifact of recent fixes, or an

    2. The relationship between the separate windows and the browser back button is confusing. My interaction went like this:

      1. Entered name schedule name "Winter 2012".

      2. Got new window with name "Winter%202012".

      3. Went back initial window and hit browser Refresh button, in hopes that the new schedule document would show up in the list.

      4. Instead, pressing the browser Refresh took me to the Login page.

      5. I logged in again, and saw to copies of "Winter 2012" in the document list. Not sure what just happened.

      6. So I delete both document and try to re-create with name "Winter 2012" and I get "Error: Schedule named Winter 2012 already exists ...". This is TOTALLY ANNOYING.

      7. So I create "Winter 2012, Again"

    3. The point of the preceding scenario was to try to get rid of the "%20" characters in the schedule names, which I see at this point is just a bug that needs to be fixed.

    4. There also appear to be extra characters in the name that stayed there when I pressed the 'Delete' button while entering the schedule name originally.

    5. Also, I think 'Trash' should be to the far right, not right next to 'Home'. The point is that 'Trash' should have some visual separation from the non- trash.

    6. If the 'My Scheduling Documents' window is going to stay up, it needs to auto- refresh the list of schedule documents when new ones are created, not just saved. To distinguish in the list between saved and unsaved documents, there can be some form of standard marker indicating that a document has unsaved changes.

    7. With further interaction, I see that it refreshes when I press 'Home'; I'd still like auto-refresh-on-create, and an 'unsaved changes' icon for each document with unsaved changes.

    In 'Trash' page:

    1. There needs to be an 'Empty Trash' button at the bottom.

    2. I suspect that the reason that I can't reuse the name "Winter 2012" is that it's in the trash; this should not be the case. What's particularly confusing in the current state is that there are two separate documents in the trash both named "Winter 2012", and I cannot name a non-trash document "Winter 2012".

    3. I'm still not sure that this page is sufficiently intuitive or useful. I might prefer an iTunes style idea for trash, where when the user deletes a schedule document, the program asked whether or not the document file should be deleted?

    4. This allows the normal file system to manage the trash. It does of course require that files actually be created for schedule documents, which I would not mind at all.

    5. Need spacing between list and 'Restore ...' button at the bottom of the page.

    Pressing Logout Button:

    1. It's confusing to me to have two separate pages both logged out.

    2. In my opinion, this is one case of many where a web-based application is inferior to a desktop application.

    Pressing a schedule document link:

    1. It's a bit confusing having the check box next to the name, in that it's not initially clear if I need to check it to work on the document..le

    2. I supposed this confusion will go away after some use.

    Miscellaneous:

    1. After I close the 'Home' window, I don't' see a way to get back.

    2. The 'Settings' menu should be on the far right.


2. Enter Resource Data

    General:

    1. As on other pages, there needs to be spacing around buttons.

    2. I think the top 'New' button should be removed, since the one at the bottom is sufficient.

    3. The 'updateheadersidths' button needs to be removed.

    4. At present, I don't see any way to delete entries from any of the resource pages.

      1. We've discussed an number of alternatives to make this happen, the most recent of which was allowing one or more rows selectable, with a 'Delete' button on the bottom of the page.

      2. This is a critical issue that needs to be fixed before client deployment.

    5. I find the 'Successfully Saved' pop-up increasingly annoying after a while, particularly when it pops up after several seconds and I'm in the middle of typing.

    6. Also, I inadvertently typed 'Command-S' to save and got the Browser's save dialog; I think this may confuse some users.

    7. In Firefox:

      1. When single clicking, edit typing bar does not appear consistently in edit boxes. Need to double click to get typing bar in the correct place.

      2. When typing tab, edit typing bar goes one past where it should be.

    8. Copy and paste of rows would be very handy.

    Courses:

    1. MAJOR ERROR 1:

      1. After entering six rows, but before saving, I selected the small vertical line with 3 vertical dots on the left of a row and then hit the 'Delete' key; the intent was to try to delete an empty row.

      2. The program logged me out and put me back at the 'Login' page.

      3. All unsaved data were lost.
      MAJOR ERROR 2:

      1. While entering about the tenth course, I pressed the delete key in the normal way within a data entry box and I got this error: 
        There was an error getting the schedules: 500 The call failed on the server;
see server log for details.

      2. The entire document appears to be lost.

    2. Evidently there is no uniqueness criterion for courses.

      1. I entered separate rows, with the same values for all data fields and it saved OK.

      2. This should not be possible.

      3. That is, I suggest that the uniqueness criterion should be that no to courses can have exactly the same values for all data fields.

    3. 'Department' heading should be 'Prefix'.

    4. 'Catalog Number' heading should be 'Number'.

    5. In Firefox, the drop-down in the 'Course Type' is difficult to work with, in particular 'click-release' does not work to select the down arrow, but instead requires 'click-and-hold'.

    6. In Firefox, I could not get the drop-down under 'Associations' to select anything other that '(none)'. [UPDATE: it worked after more than one course was entered; I think the initial error is reproducible.]

    7. WTU and SCU Conceptual Issues:

      1. I believe the WTU value should have a defaults entered as follows: 4 for a 3-hour lecture, 5 for a 4-hour lecture, 1 for a 3-hour lab.

      2. There needs to be some way to account for variable WTU values; it may be that we just enter multiple rows of the same course, but I'd prefer to be able to input a WTU range in one row, as in "1-4".

      3. I don't think we need the SCU column for scheduling purposes; I suggest we remove it.

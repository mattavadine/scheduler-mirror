Week 6 Acceptance Test Report for the IME Department

These are the results from both Kaylene's acceptance test and the feedback that I got from my roommates from acceptance testing.



1. Create, Save, and Re-Open a New Schedule

This works.


2. Enter Resource Data

  1. DayCombos bug. See jira issue #495, #498, #496 Also, day combos don't show up after the first row

  2. It's not clear to the user whether or not max enrollment was per section or overall for the entire course

  3. They viewed the "Schedulable" check box as a "Selection" text box instead of the blue highlight being row selection (but they quickly figured that out when checking it to delete it didn't work)


3. Generate a Schedule

  1. We need to get rid of the "debug" message that tells how long the courses take to load.

  2. When I tried to press the generate button I got an error: "Failed to get schedule: com.google.gwt.user.client.rpc.StatusCodeException: 500 The call failed on the server; see server log for details"

  3. They didn't like that the Course Preferences Box wasn't around the course preferences (it is off kilter)

  4. Also, by default they put @calpoly.edu at the end of the username (this might be because I gave them faculty email addresses, but it must be more clear)

  5. One of them wasn't sure what to do initially to be able to enter resource data after she pressed "add course" because the first cell isn't in an "editable" state, it took her a second to realize that she had to click where the cell is to make it editable


4. Edit the Schedule

  1. They didn't like that Generate spit out "gross looking" schedules with lots of classes in a single time spot

  2. When editing, by default it should give a start time and and end time that amounts to an hour difference, not a 1/2 hour difference.


5. Save, Exit, Re-Open Schedule

  1. This seems to work.


6. Quality Measures

  1. The resource tables are getting to a stable and usable place; however, work needs to be done to fix the minor bugs, and to make sure that data validation is occuring, CORRECTLY at this point.

  2. The generation page is not acceptable. I cannot press the generation button, I still get errors. And when I try to drag courses onto the schedule, it doesn't seem to be running the algorithm to assign instructors or rooms to these courses.

  3. The top-level UI functionality doesn't seem to be working, specifically the File, Open menu.


7. Other Issues

  1. Generate still fails on invalid data errors.

  2. The DayCombos in the resource table aren't accurate.

Initial Report on Generating Winter 2012 Schedule for GRC Department



Entering Resource Data

There seems to be an issue creating labs. I made GRC 201, and another GRC 201 with the same course name as well, and it complained because they were identical. I then changed the course name for the lab section to end in "lab", and it stopped complaining. However, when I reload the page, the lecture section is gone. Apparently there was an error on the back end that never made it to the user, so the user continues thinking it got saved correctly.

Course preferences do not work, either they aren't being saved or they aren't being loaded correctly, because it always reverts to the default.


Generating a Schedule

The generated schedule I had put everything at 7-8am. On top of that, it was ridiculously slow. That was only when it did work. I used to get 500 errors, though I can't reproduce it now.

It did appear to contain all of the classes from the courses tab. However, I couldn't enter a lot of the courses into the courses tab, so I suppose we don't know if the scheduling view will work with those courses.

I didn't see any scheduling conflicts.


Editing the Schedule

When it isn't giving me 500 errors, dragging and dropping is incredibly slow. Other than that, it worked fine.


Other Issues

Many other smaller issues, see the JIRA.

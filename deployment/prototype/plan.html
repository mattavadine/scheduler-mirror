Deployment Plan

The Scheduler project may be deployed in one of two deliverables: a web-based application and a desktop application. The current plan is to move forward with the development of each app, and test the various methods of deployment. The prototype team is also looking into the possibility of developing one application that supports both web and desktop applications.


Web-Based Application

Deployment process requires a supported web browser and login credentials. A user would follow the link to web app and have access to everything in the prototype.

Pros

  • Simple deployment process: as long as the user has a supported browser, it will work as expected.
  • Available anywhere without the installation of software other than a browser.
  • Updates are automatically applied to the site without any new installation required of the user.

Cons

  • Requires active connection to the internet at the time of use.
  • Need to determine which browsers to support, possibly supporting at least 3 different browsers: Firefox, Internet Explorer, and Chrome.
  • Updated versions of browsers may require more maintenance of the code.


Desktop Application

Deployment process includes the installation of a .jar or .exe for the client to run. This process may need to include installation of java and any external server software.

Pros

  • More control over the environment the customer will use the program in.
  • Installation process can include java and SQL installations to ensure successful product deployment.

Cons

  • Need to install a supported version of java.
  • Updates require delivery of a new executable file and installation process.

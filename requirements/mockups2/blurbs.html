old Login

When the user launches the scheduling program, the program displays its start-up screen. The commands along the left of the screen are the main functions of the program. The Home screen is where the scheduling process begins. The Schedule commands allow the user to to Build, View, and Edit a schedule. The Manage commands allow the user to create and edit information for Instructors, Courses, and course Locations. To get to the Home screen, the user logs in.

old Home (Select a Term)

After successful login, the program presents the Home screen. The user first selects the quarter and year to generate a scheduler for. Then the user can either select a previous quarter as the basis for the new schedule, or generate a new schedule from scratch. In this case, let's suppose the user presses the New Schedule Button.

stewart Schedule Generation (Blank)

After opening up a New Schedule, the user is presented with a blank calendar view and a list of all possible department courses (shown on the left). Courses can be dragged from the Department Courses list directly onto the calendar, or into the Autofill Courses list.

stewart Schedule Setup (Populated)

Once a user has dragged courses into the Autofill Courses section, the list of courses can be placed in the schedule by clicking on the Auto Fill button. This will schedule all courses in the Autofill Courses list, and take into account any classes that already exist on the calendar so that no conflicts occur.

stewart Schedule Setup (Accepted)

When a user drags a course onto the calendar, the program verifies that there aren't any conflicts with the proposed schedule. If the schedule does not contain any conflicts, the course is added to the calendar. In this instance, CHEM 252 has been added to the schedule. The course is colored light grey to represent that the scheduled time is preferred based on the preferences of an instructor.

stewart Schedule Setup (Not Preferred)

In this case, the user has scheduled CHEM 590 in a time that is not preferred due to the preferences of an instructor. CHEM 590 is colored dark gray to represent this.

stewart Schedule Setup (Hover)

The user can move the mouse over courses to see additional information. Here the user has rolled over CHEM 461, Section 1.

stewart Schedule Setup (Drag Select)

If a course can be offered over different sets of days during the same time slot, a window pops up that asks the user to pick the desired schedule for the course. Here the user has chosen to offer CHEM 474 on Monday, Wednesday, and Friday.

stewart Schedule Setup (Drag Select Alternate)

In this case the user has offered CHEM 474 on Monday, Wednesday, and Thursday instead of Monday, Wednesday, and Friday.

chang Course Manage (Basic)

When the user selects 'Courses' in the main menu, the program displays an overview of the courses offered by the department.

chang Course Manage (Add)

To add a course, the user enters data in the bottom row, and presses the 'Add' button.

chang Course Manage (Edit Row)

To edit a course, the user selects a row in the table. The program highlights the editable fields, where the user enters the desired new data.

chang Instructor List

We are now going to examine features in the Manage section of the main menu.

When the user selects 'Instructors' in the main menu, the program displays an overview of the instructors in the department.

chang Instructor Remove

When a user presses the 'Delete' button, this confirmation window will popup, verifying that the user wants to remove the instructor from the schedule.

chang Instructor Edit

To edit an instructor, the user selects the instructor's name and presses the 'Edit' button. The program highlights the editable fields, where the user enters the desired new data. The program also adds a row of buttons below the instructor's information. These buttons provide access to instructor preferences for Courses, Times, and Distance between classes.

chang Instructor Prefs (Courses)

When the user presses the 'Courses' button on the preceding screen, the program displays this dialog. For a user acting as department scheduler, the edits can be performed for any instructor. When individual instructors run the scheduling program, they set their own preferences. The instructor-set preferences are the default values used for scheduling. Note that a department scheduler can only edit preferences for a particular schedule, not the instructor's permanent stored preferences.

chang Instructor Prefs (Times)

When the user presses the 'Edit Preferences: Time' button, the program displays this dialog. As with course preferences, the edits can be performed by two different categories of users. Department schedulers edit time preferences only for a particular working schedule. Instructors edit their own preferences, which are the default values unless changed by the department scheduler for a particular quarter.

chang Instructor Prefs (Distance)

The user defines an instructor's distance preference by specifying a reference building on campus, and the maximum distance from that building that the instructor would prefer to travel. When generating a schedule, the program will try to assign the instructor's courses to rooms that are within the given distance from the reference building.

chang Database Room (Main)

When the user selects 'Locations' in the main menu, the program displays an overview of the locations available to the department for scheduling classes.

chang Database Room (Add)

To add a location, the user enters data in the bottom row, and presses the 'Add' button.

chang Database Room (Edit)

To edit a course, the user selects a row in the table. The program highlights the editable fields, where the user enters the desired new data.

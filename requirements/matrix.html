For the Week 5, Winter 2012, release, we will incorporate the following features:

Below is a list of the more general features that were established at the start of the project:

During initial phase one interviews, each department was asked to explain their scheduling procedure and suggest features and tools that would be useful to them. After interviewing all departments participating in the project, the following feature matrix was designed to reflect the overall importance of the program's features.

Each feature is ranked on a scale of -2 to 2, representing the following:

  •  2: This feature is mandatory. The program is not considered functional without this features.
  •  1: This feature is a priority. Most departments have expressed an interest in this feature and it should be supported.
  •  0: This feature is optional. Certain departments have mentioned this feature as a nice add-on, but it is not neccessary.
  • -1: This feature is not important. It was mentioned, but contributes little to the functionality of the program.
  • -2: This feature is not useful. It was mentioned, but is not appropraite to the goals of tool.

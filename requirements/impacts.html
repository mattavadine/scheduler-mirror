1.4 Impacts

This section addresses the potential impacts of implementing the scheduling software.

1.4.1 Distribution Problems

The software will be distributed in the form of an installation onto a department computer. This department computer will then act as the backbone for the software for use with that department. Each department using this software will need their own department computer dedicated to the task of being the software backbone. A problem with this setup is that when there is any upgrade available, a human is required to manually upgrade the software on the department computer.

1.4.2 Installation Problems

If a department does not have a computer available to be used for the scheduler software, then there will be a problem of not having a computer to install onto.

1.4.3 Regular Usage Problems

Responsibility of keeping the software secure and out of reach of unauthorized people will be up to the departments themselves. Security will be a regular problem for the users. There may be a policy of never leaving unused computers turned on. In the case of the department computer dedicated to the scheduler software, it must be kept on.

1.4.4 Departmental Impacts

The positive and negative impacts the Scheduler would have on departments.

1.4.4.01 AERO

Benefits: The scheduling process would be streamlined and would make finding/fixing conflicts easier.

Disadvantages: None

1.4.4.02 BUS

Benefits: The process of constraint checking would be eased, as would making changes after the initial schedule is generated.

Disadvantages: None.

1.4.4.03 CENG

Benefits: The process of constraint checking would be eased, as would making changes after the initial schedule is generated.

Disadvantages: None.

1.4.4.04 CHEM

Benefits: The process of constraint checking would be eased, as would making changes after the initial schedule is generated.

Disadvantages: Time to learn new Scheduler and adaption to new scheduling process.

1.4.4.05 CM

Benefits: Data entry would be easier than current process.

Disadvantages: Time to learn new Scheduler and adaption to new scheduling process.

1.4.4.06 EE

Benefits: Eases gathering teacher preferences and dealing with schedule changes/conflicts

Disadvantages: None

1.4.4.07 ENGL

Benefits: Eases gathering of teacher preferences and overall time spent on scheduling.

Disadvantages: None

1.4.4.08 FSN

Disadvantages: None.

1.4.4.09 GRC

1.4.4.10 IME

Benefits: Speed up the gathering of teacher preferences

Disadvantages: None.

1.4.4.11 JOUR

Benefits: Allows teachers to view the completed schedule but not edit it.

Disadvantages: None

1.4.4.12 MATH

Benefits: Ease collaboration between those who work on the schedule.

Disadvantages: None.

1.4.4.13 MU

Benefits: Initial scheduling would be sped up, as would finding and dealing with constraint issues.

Disadvantages: None

1.4.4.14 PHYS

Benefits: : Would make the department scheduler's job easier, particularly for entering data into Peoplesoft.

Disadvantages: None

1.4.4.15 POLS

Benefits: Automates the gathering of instructor preferences, minimizing scheduling conflicts and minimizes the number conflicts the department chair would have to resolve.

1.4.4.16 PSY

Benefits: Data entry would be simplified.

Disadvantages: None

Benefits: Time would be saved due to ability to use past schedules as bases and the drag and drop interface for quickly modifying a schedule.
